计算机网络复习

第一章：网络基础

  1. 网络体系结构
    • 分层特点：分解问题，模块化设计，提供接口。越多不利于整体优化，越少单层难度越大。
    • 模型对应关系
  2. 吞吐量，带宽，延迟，抖动，丢包

第二章：网络连接

  • 即网络接口层所对应的事务（OSI：数据链路以及物理层）
    • 物理层：基础IEEE协议
    • 数据链路：数据编码（NRZ，NRZI，曼彻斯特，4B/5B等），差错控制校验，字节字层比特组帧协议，以太网无线局域网连接

第三章：网络互联

  1. 连接基础
    • 交换机工作在链路层，其转发分组方式分为：
      1. 数据报：无连接，维护转发表，不确定是否可以接收
      2. 虚电路：建立VCI标识符，有连接，明确路径，降低首部开销，（ATM异步传输模式）
      3. 源路由：源地址进行路由选择而非沿途路由器
    • 网桥：将网络的多个网段在数据链路层（OSI模型第2层）连接起来（即桥接）
      • 生成树协议（Spanning Tree Protocol，STP）
      • 局限性
  2. 互联网基础
    • 服务模型：
      • IP数据报，在其经过的物理网络上进行重新封装与重组
      • 全局地址：A类（0+7位），B类（10+14位），C类（110+21位）
      • 子网划分与CIDR（无类域间路由），IP转发规则基于“最长匹配”
      • 地址解析协议（ARP）：IP地址与物理地址的转化
      • 动态主机配置协议（DHCP）：客户端向服务器请求地址
      • 差错报告（ICMP）：向源主机发送的错误报告以及查询
      • 虚拟专用网（VPN）：避免X与Y公司的直接连接，而是从一个专用网络收发
  3. 路由
    • 路由与转发区别：转发是动作，路由是建立转发表的过程
    • 路由信息协议（RIP）：
      • 扩散方式，不断向周围广播自己路由表并自动更新路由表；
      • RIP协议处于UDP协议的上层，RIP所接收的路由信息都封装在UDP协议的数据报中，RIP在520号UDP端口上接收来自远程路由器的路由修改信息，并对本地的路由表做相应的修改，同时通知其它路由器。
      • 最大跳数15
    • 开放式最短路径优先（OSPF）：
      • 过程：
        • 发Hello报文
        • 建立邻接关系，扩散LSP（链路状态分组）
        • 形成链路状态数据库
        • SPF算法（Dijkstra最短路径）
        • 形成路由表
      • SPF算法：依次访问下一节点的LSP，并将最小的开销记录计入到证实表中
    • 交换机与路由器实现

第四章：高级网络互联

  1. 边界网关协议（BGP）
    • AS号唯一，避免闭环
    • AS内采用域内协议（RIP，OSPF）
    • BGP制定策略而非写入协议，例如人为控制流量导向或者拒绝为其他AS过度流量
    • BGP路径选择
  2. IPv6
    • 32位拓展到128位
    • 受益于CIDR，其不分类
    • 分组首部更简洁
    • 自动配置
    • 路由首部
  3. 多播
    • 与单播比较，由发送n次变为发送1次
    • 与广播比较，组员可以自行选择是否像组内路由器复制数据，广播是强制性的
    • 多播路由
      • DVMRP（距离矢量组播路由选择协议）：Distance Vector Multicast Routing Protocol，基于RIP
      • PIM-SM（稀疏模式独立组播协议）：选出一个RP，传过来的数据要么直接给RP，要么由组员通过隧道传给RP。其是域内协议
      • MSDP（多播源发现协议）：域间协议，将各个RP汇聚在一起
      • PIM-SSM：已经知道了组播源的位置，就不再需要汇集点与生成树
      • BIDIR-PIM（双向PIM）：多对多，并且不需要RP
  4. MPLS（多协议标签交换）
    • 利用标签引导数据高速、高效传输；不仅仅支持网络层，链路层协议也支持
    • 实现原理
      • 入口LER粘贴标签
      • 通过传统路由协议建立连接
      • 根据相同标签LSR进行相应调整
      • 出口LER撕掉标签
    • 显示路由：由于有了标签，可以相应自己挑选流量路径
    • VPN隧道
  5. 移动IP

第五章：端到端协议

  1. UDP：无连接，无保证，有长度，支持多播广播;<主机，端口>对构成了UDP协议的多路分解密钥
  2. TCP:面向连接，可靠的
    • 两种控制区别
      • 流量控制：防止发送方数据超出接收方接收能力
      • 拥塞控制：防止过多数据注入网络而造成交换机链路超载
    • 三次握手：
      • 客户端发送序号x的SYN
      • 服务端发送序号y的SYN以及序号x+1的ACK
      • 客户端发送序号y+1的ACK
    • 四次握手
      • 客户端发送序号x的FIN
      • 服务器发送序号x+1的ACK
      • 服务器发送序号y的FIN
      • 客户端发送序号y+1的ACK
    • 核心算法：滑动窗口，由接收方控制窗口大小，发送方根据ACK依次移动窗口左边界
      • 发送方：LastByteAcked，LastByteSent，LastByteWritten
      • 接收方：LastByteRead，LastByteExpected，LastByteRcvd
        • 缓冲区大小=LastByteRcvd-LastByteRead
        • 通知窗口大小=缓冲区大小-（LastByteExpected-LastByteRead）
      • 发送方的Sent-Acked必须小于通知窗口，其和通知窗口的差值为此时的有效窗口
    • 傻瓜窗口症状：一次报文长度过小
    • 自适应重传：
      • 根据RTT估计RTO（重传超时时间）

  3. 远程过程调用RPC

    • 一个计算机通信协议，用于不同计算机之间的函数调用，集群部署

  4. 实时应用程序重传（RTP）

    • 传输层之上，应用层之下，UDP
    • 实时数据压缩处理，给RTP封装成UDP数据报，再给IP封装
    • 控制由RTCP命令
    • 端口选择，RTP为偶数端口，RTCP为接下来的奇数端口

第六章：拥塞控制

  1. 概述
    • 资源分配与拥塞控制
      • 资源分配：一个竞争过程以满足对网络资源的竞争需求
      • 拥塞控制：防过载
    • 分类方法
      • 主机（协议解决）/路由（网络内部解决）
      • 预定（提前解决）/反馈（有现象再解决）
      • 窗口（TCP）/速率（视频传输）
  2. 排队规则
    • FIFO（带队尾丢弃）
    • 公平排队
  3. TCP拥塞机制
    • 加性增/乘性减
    • 慢启动：到达拥塞窗口之前，指数增长
    • 快速重传：连续三次接到相同ACK
    • 快速恢复：去除了慢启动
  4. 拥塞避免
    • 依赖路由器的拥塞避免
      • DECbit拥塞位
      • RED随机早期检测：设置阈值，超出能力后进行相应处理；实际应用中双阈值防抖动
  5. 服务质量
    • 实时应用分类
      • 弹性型
      • 实时型
        • 非容忍：机器人
        • 容忍
          • 非自适应
          • 自适应
            • 速率可适应
            • 延迟可适应
    • RSVP
      • 流说明
        • RSpec:指定延迟目标或界限
        • TSpec:指定带宽
      • 容许控制：查看RSpec和TSpec以确定是否容许
      • RSVP资源预留协议
        • 不应损害当今无连接网络的健壮性
        • 支持单播多播
        • 依次向上路由进行资源预留，其本身不是路由协议
      • 区分服务DSCP
        • EF：加速转发，提供延时保证
        • AF：确保带宽，用于带宽保证，参数设置影响丢弃概率，基于RED

第八章：网络空间安全

  1. 加密标准

    • DES——3DES——AES（高级加密标准）
      • ECB
      • CBC
      • CFB
      • OFB
    • 摘要加密算法
      • MD5（Message Digest 5）——128位摘要
      • SHA-1——160位摘要
      • SHA-2/3
    • DSS（数字签名标准）
      • RSA
      • ElGamal
      • 椭圆曲线数字签名算法

  2. 密码基础

    • 对称密钥：加密解密密钥相同
    • 公钥密码：私钥拥有者可以解密信息
    • 认证码：
      • 信息摘要，解密明文后计算的摘要和直接解密出的摘要对比
      • MAC值，以双方均知道的一个秘密值为参数计算MAC
  3. 密钥预分发
    • 公钥基础设施（PKI）：
      • 认证机构CA
      • 良好网络PGP
      • CRL证书撤销列表
    • 对称密钥分发
      • 密钥分发中心（KDC）
  4. 认证协议
    • 原始性：时间戳
    • 公钥认证
      • 协议1：A发签名和时间，B发签名和时间和密钥
      • 协议2：A发签名和时间，B发A的时间发签名和时间，A发B的时间和密钥
    • 对称密钥
      • 通过KDC作为认证服务器AS
      • Diffie-Hellman密钥协商（私自保有a,b值，无需预分发）
  5. 实例
    • PGP——邮件安全
    • SSH：初次连接无法有效认证
      • SSH-TRANS
      • SSH-AUTH
      • SSH-CONN
    • 传输层
      • TLS
      • SSL
      • HTTPS
    • IP安全
      • IPsec -无线安全
      • 802.11i
        • WEP
        • WPA2
        • EAP
        • CCMP
          • CBC-MAC
    • 防火墙
